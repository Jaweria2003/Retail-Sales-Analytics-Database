
MY PROJECT- Retail Sales Analytics Database
- Analyze trends, customer behavior, and product performance.
Designed and implemented a relational database to analyze retail business performance, customer behavior, and product trends. Built tables for customers, products, stores, and sales transactions, and performed advanced SQL analysis to extract actionable insights.

Key Contributions:

- Designed normalized schema with 4 core tables: customers, products, stores, and sales.

- Loaded and cleaned retail sales data using SQL Developer.

- Performed analytical queries to identify:

- Top-selling products

- Monthly/Quarterly revenue trends

- High-value customer segments

- Store-wise performance insights

- Used JOINs, CTEs, Aggregate & Window functions for analysis.

- Created views for automated reporting.

Skills Shown: SQL(Joins, Aggregations, CTEs, Window Functions)
Tools: Oracle SQL Developer


--- Customer Table ---
Create Table customers (
customer_id Number PRIMARY KEY,
customer_name VARCHAR(30),
gender VARCHAR(10),
city VARCHAR(20),
state VARCHAR(50)
);

--- Product Table ---
Create Table products (
product_id NUMBER PRIMARY KEY,
product_name VARCHAR(30),
category VARCHAR(30),
price NUMBER(10,2)
);

--- Stores Table ---
CREATE TABLE stores (
store_id NUMBER PRIMARY KEY,
store_name VARCHAR(30),
city VARCHAR(20),
state VARCHAR(30)
);

--- Sales Table ---
CREATE TABLE sales (
sales_id NUMBER PRIMARY KEY,
customer_id NUMBER references customers(customer_id),
product_id NUMBER references products(product_id),
store_id NUMBER references stores(store_id),
sale_date DATE,
quantity NUMBER,
total_amount NUMBER(10,2)
);
